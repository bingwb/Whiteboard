在数据库（DB）领域，日本现场除了使用 SQL 标准术语外，还有很多特有的“日本式”表达（比如把 Table 称为“表”或把 Record 称为“件”）。

🗄️ DB <ruby>関連<rt>かんれん</rt></ruby><ruby>常用<rt>じょうよう</rt></ruby><ruby>用語集<rt>ようごしゅう</rt></ruby>

1. <ruby>基本要素<rt>きほんようそ</rt></ruby> - 基础对象

| 术语 | 读音 | 说明 |
|---|---|---|
| <ruby>表<rt>ひょう</rt></ruby> / テーブル | テーぶる | 数据表。文档里常写成「〇〇表」。 |
| <ruby>列<rt>れつ</rt></ruby> / カラム | からむ | 列 / 字段。 |
| <ruby>行<rt>ぎょう</rt></ruby> / レコード | れこーど | 行 / 记录。现场常说「1<ruby>件<rt>けん</rt></ruby>（いっけん）」。 |
| <ruby>主キー<rt>しゅきー</rt></ruby> | PK / しゅきー | Primary Key（主键）。 |
| <ruby>外部<rt>がいぶ</rt></ruby>キー | FK / がいぶきー | Foreign Key（外键）。 |
| <ruby>索引<rt>さくいん</rt></ruby> / インデックス | いんでっくす | Index（索引）。 |
| <ruby>型<rt>かた</rt></ruby> / データ<ruby>型<rt>がた</rt></ruby> | でーたがた | 数据类型（如 VARCHAR, INT 等）。 |
| <ruby>制約<rt>せいやく</rt></ruby> | せいやく | Constraint（约束，如 NOT NULL, UNIQUE）。 |

2. SQL<ruby>操作<rt>そうさ</rt></ruby> - 常用操作
 * <ruby>抽出<rt>ちゅうしゅつ</rt></ruby> (Select): 提取/查询。常说：「データを<ruby>抽出<rt>ちゅうしゅつ</rt></ruby>してください」。
 * <ruby>更新<rt>こうしん</rt></ruby> (Update): 更新。
 * <ruby>挿入<rt>そうにゅう</rt></ruby> (Insert): 插入。
 * <ruby>削除<rt>さくじょ</rt></ruby> (Delete): 删除。
 * <ruby>物理削除<rt>ぶつりさくじょ</rt></ruby> vs <ruby>論理削除<rt>ろんりさくじょ</rt></ruby>: * 物理删除（硬删除，直接 DELETE）。
   * 逻辑删除（软删除，通过 is_deleted 标志位更新）。日本现场极度常用。
 * <ruby>紐付<rt>ひもづ</rt></ruby>け (Join/Link): 关联。常说：「AテーブルとBテーブルを<ruby>紐付<rt>ひもづ</rt></ruby>ける」。
 * <ruby>洗替<rt>あらいがえ</rt></ruby> (Replace/Overwrite): 全删全插。指先清空表数据再重新导入。

3. <ruby>設計・管理<rt>せっけい・かんり</rt></ruby> - 设计与管理
 * <ruby>正規化<rt>せいきか</rt></ruby>: 范式化 / 数据库规范化。
 * <ruby>属性<rt>ぞくせい</rt></ruby>: Attribute / 属性。
 * <ruby>排他制御<rt>はいたせいぎょ</rt></ruby>: 并发控制/锁（乐观锁/悲观锁）。
 * デッドロック: Deadlock（死锁）。
 * <ruby>実行計画<rt>じっこうけいかく</rt></ruby>: Execution Plan（执行计划/解释计划）。
 * <ruby>移行<rt>いこう</rt></ruby>: Data Migration（数据迁移）。

4. <ruby>現場<rt>げんば</rt></ruby>でよく<ruby>使<rt>つか</rt></ruby>われるフレーズ - DB 相关常用语
 * 「<ruby>本番環境<rt>ほんばんかんきょう</rt></ruby>でSQLを<ruby>直打<rt>じかうち</rt></ruby>しないでください。」
   * (请不要在生产环境直接手打 SQL 运行。) —— 必须要准备好脚本并经过审核。
 * 「このカラムにインデックスを<ruby>貼<rt>は</rt></ruby>ってください。」
   * (请给这个字段加上索引。) —— 这里的“加”习惯用「<ruby>貼<rt>は</rt></ruby>る（贴）」。
 * 「データが<ruby>不整合<rt>ふせいごう</rt></ruby>を起こ（おこ）しています。」
   * (数据发生了不一致/不整合。)
 * 「<ruby>大量<rt>たいりょう</rt></ruby>データなので、パフォーマンスに<ruby>懸念<rt>けねん</rt></ruby>があります。」
   * (因为是海量数据，担心会有性能问题。)
 * 「<ruby>念<rt>ねん</rt></ruby>のため、バックアップを<ruby>取得<rt>しゅとく</rt></ruby>しておいてください。」
   * (为了保险起见，请预先取得备份。)

5. <ruby>書類<rt>しょるい</rt></ruby>・ツール - 文档与工具
 * ER<ruby>図<rt>ず</rt></ruby>: Entity-Relationship Diagram。
 * テーブル<ruby>定義書<rt>ていぎしょ</rt></ruby>: 数据库表定义文档（通常是 Excel 格式）。
 * CRUD<ruby>図<rt>ず</rt></ruby> / CRUD<ruby>表<rt>ひょう</rt></ruby>: 增删改查对照表。
 * ダンプ (Dump): 导出数据备份文件。
 * <ruby>流<rt>なが</rt></ruby>し<ruby>込<rt>こ</rt></ruby>み: 导入数据。常说：「SQLファイルを<ruby>環境<rt>かんきょう</rt></ruby>に<ruby>流<rt>なが</rt></ruby>し<ruby>込<rt>こ</rt></ruby>む」。

💡 现场特别注意：数据件数
日本现场对“件数”非常敏感。
 * 运行完 UPDATE 或 DELETE 后，Leader 一定会问你：
 * 「<ruby>何件<rt>なんけん</rt></ruby>、<ruby>更新<rt>こうしん</rt></ruby>されましたか？」 (更新了多少条数据？)
 * 此时必须给出精确数字，并附上带有 「◯ row(s) affected」 的截图作为证据（Evidence）。
