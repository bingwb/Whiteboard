在 Jira 或 GitHub 上，沟通通常更倾向于简洁、明确，并且由于是书面交流，使用正确的「敬语/谦逊语」能让协作更顺畅。
以下是为您整理的常用注释模版，同样使用了 <ruby> 标签标注假名。

📝 Jira / GitHub <ruby>常用<rt>じょうよう</rt></ruby>コメント<ruby>集<rt>しゅう</rt></ruby>
1. GitHub: プルリクエスト (PR) <ruby>作成時<rt>さくせいじ</rt></ruby>
> 提交 PR 时，清晰描述“做了什么”和“请看哪里”。
> 
 * 【<ruby>概要<rt>がいよう</rt></ruby>】〇〇<ruby>機能<rt>きのう</rt></ruby>の<ruby>追加<rt>ついか</rt></ruby>
   * (【概要】添加了 XX 功能)
 * 【<ruby>対応内容<rt>たいおうないよう</rt></ruby>】バグの<ruby>修正<rt>しゅうせい</rt></ruby>およびリファクタリングを<ruby>行<rt>おこな</rt></ruby>いました。
   * (【对应内容】进行了 Bug 修复及重构。)
 * ご確認（<ruby>かくにん</rt></ruby>）をお<ruby>願<rt>ねが</rt></ruby>いします。
   * (请帮忙确认。)
 * お<ruby>忙<rt>いそが</rt></ruby>しいところ<ruby>恐縮<rt>きょうしゅく</rt></ruby>ですが、レビューをいただけますでしょうか。
   * (百忙之中非常抱歉，能请您 Review 一下吗？)
2. GitHub: コードレビューへの<ruby>返信<rt>へんしん</rt></ruby>
> 被指摘（Review 指出问题）后的回复。
> 
 * ご<ruby>指摘<rt>してき</rt></ruby>ありがとうございます。
   * (谢谢您的指出。)
 * ご<ruby>指摘<rt>してき</rt></ruby>の<ruby>通<rt>とお</rt></ruby>り<ruby>修正<rt>しゅうせい</rt></ruby>しました。ご確認（<ruby>かくにん</rt></ruby>）をお<ruby>願<rt>ねが</rt></ruby>いします。
   * (正如您所指出的，我已经修改了。请确认。)
 * こちらの<ruby>実装<rt>じっそう</rt></ruby>にした<ruby>意図<rt>いと</rt></ruby>は、〇〇のためです。
   * (采用这种实现方式的意图是出于 XX 的考虑。)
 * <ruby>修正<rt>しゅうせい</rt></ruby>が<ruby>完了<rt>かんりょう</rt></ruby>したため、<ruby>再<rt>さい</rt></ruby>レビューをお<ruby>願<rt>ねが</rt></ruby>いします。
   * (修正已完成，请再次 Review。)
3. Jira: <ruby>進捗<rt>しんちょく</rt></ruby>ステータスの<ruby>更新<rt>こうしん</rt></ruby>
> 在 Ticket（票据）上更新任务状态。
> 
 * 【<ruby>進捗<rt>しんちょく</rt></ruby>】<ruby>現在<rt>げんざい</rt></ruby>、テストを<ruby>実施中<rt>じっしちゅう</rt></ruby>です。
   * (【进度】现在正在进行测试。)
 * 【<ruby>報告<rt>ほうこく</rt></ruby>】<ruby>本日中<rt>ほんじつちゅう</rt></ruby>にプルリクを<ruby>出<rt>だ</rt></ruby>す<ruby>予定<rt>よてい</rt></ruby>です。
   * (【报告】预计今天内会发出 PR。)
 * 〇〇の<ruby>影響<rt>えいきょう</rt></ruby>で、<ruby>作業<rt>さぎょう</rt></ruby>に<ruby>遅<rt>おく</rt></ruby>れが<ruby>発生<rt>はっせい</rt></ruby>しています。
   * (受 XX 的影响，作业出现了延迟。)
4. Jira: <ruby>質問<rt>しつもん</rt></ruby>・<ruby>相談<rt>そうだん</rt></ruby>
> 在 Ticket 下方向负责人提问。
> 
 * 〇〇について、<ruby>詳細<rt>しょうさい</rt></ruby>を<ruby>確認<rt>かくにん</rt></ruby>させてください。
   * (关于 XX，请允许我确认一下细节。)
 * このチケットの<ruby>優先順位<rt>ゆうせんじゅんい</rt></ruby>について<ruby>相談<rt>そうだん</rt></ruby>させてください。
   * (关于这张票的优先级，想跟您商量一下。)
 * お<ruby>手空<rt>てあ</rt></ruby>きの<ruby>際<rt>さい</rt></ruby>にご<ruby>回答<rt>かいとう</rt></ruby>いただければ<ruby>幸<rt>さいわ</rt></ruby>いです。
   * (如果您方便时能给予回复，我将不胜感激。)
5. <ruby>共通<rt>きょうつう</rt></ruby>: クローズ・<ruby>完了<rt>かんりょう</rt></ruby>
> 任务完成后的结束语。
> 
 * <ruby>本件<rt>ほんけん</rt></ruby>、クローズします。
   * (本事项，现在结票/关闭。)
 * <ruby>動作確認<rt>どうさかくにん</rt></ruby>がとれたため、チケットをクローズします。
   * (由于动作确认已通过，我将关闭此票。)
 * ご<ruby>協力<rt>きょうりょく</rt></ruby>ありがとうございました。
   * (谢谢您的配合。)
💡 小技巧：善用 Emoji
在日本的 Slack 或 GitHub 沟通中，为了缓和文字的生硬感，大家经常使用以下 Emoji：
 * 🙏 (:pray:)：用于请求对方、表示感谢或抱歉。
 * 👀 (:eyes:)：表示“我正在看/正在 Review”。
 * ✅ (:white_check_mark:)：表示“已完成/已确认”。
 * 👍 (:+1:)：表示“没问题/赞同”。
