既然你在使用 Azure，那么 Azure DevOps（通常简称为 AzDO）或 GitHub Actions 相关的自动化流程（CI/CD）也是必不可少的。在日本现场，关于“发布流水线”和“自动化部署”有一套非常固定的说法。
以下是为您整理的 Azure DevOps & CI/CD <ruby>関連<rt>かんれん</rt></ruby><ruby>用語集<rt>ようごしゅう</rt></ruby>。

🚀 Azure DevOps & CI/CD <ruby>常用<rt>じょうよう</rt></ruby><ruby>用語集<rt>ようごしゅう</rt></ruby>

1. <ruby>基本<rt>きほん</rt></ruby>サービス - 基础服务名称
| 术语 | 读音 | 说明 |
|---|---|---|
| Azure Repos | レポス | 源代码管理（Git 仓库）。 |
| Azure Pipelines | パイプライン | CI/CD 流水线。 |
| Azure Boards | ボーズ | 任务看板（类似 Jira）。 |
| Artifacts | アーティファクト | 编译产物/制品库。 |

2. <ruby>構築<rt>こうちく</rt></ruby>・パイプライン - 构建与流水线相关
 * ビルドが<ruby>通<rt>とお</rt></ruby>る / <ruby>落<rt>お</rt></ruby>ちる: * 编译通过 / 编译失败（挂了）。
 * セルフホステッドエージェント: * Self-hosted Agent。日本现场为了安全，经常在 VNet 内部搭建自己的 Agent。
 * <ruby>環境変数<rt>かんきょうへんすう</rt></ruby> / <ruby>変数群<rt>へんすうぐん</rt></ruby>: * Environment Variables / Variable Groups。
 * トリガー: * Trigger（触发器）。例如：「PRがマージされたら、自動（じどう）でトリガーされます」。
 * <ruby>承認<rt>しょうにん</rt></ruby>ゲート: * Approval Gates。发布到生产环境前，需要负责人点击“Approve”。

3. IaC (<ruby>構成管理<rt>こうせいかんり</rt></ruby>) - 代码化基础设施
日本 Azure 现场目前主流是 Bicep 或 Terraform。
 * ベタ<ruby>書<rt>が</rt></ruby>き: * Hard-coded（硬编码）。常被指摘：「パラメータをベタ<ruby>書<rt>が</rt></ruby>きしないでください」。
 * <ruby>定義<rt>ていぎ</rt></ruby>ファイル: * 定义文件（如 main.tf 或 main.bicep）。
 * <ruby>展開<rt>てんかい</rt></ruby> (Deploy): * 除了「デプロイ」，现场也常说「リソースを<ruby>展開<rt>てんかい</rt></ruby>する」。
 * ドライラン (Dry Run): * 预运行。在 Terraform 里叫 plan，在 Azure CLI 里叫 what-if。

4. <ruby>現場<rt>げんば</rt></ruby>でよく<ruby>使<rt>つか</rt></ruby>われるフレーズ - 自动化相关常用语
 * 「パイプラインが<ruby>異常終了<rt>いじょうしゅうりょう</rt></ruby>しました。」
   * (流水线异常终止了/报错了。)
 * 「<ruby>手動<rt>しゅどう</rt></ruby>でビルドを<ruby>実行<rt>じっこう</rt></ruby>してください。」
   * (请手动执行 Build。)
 * 「ブランチ<ruby>戦略<rt>せんりゃく</rt></ruby>について<ruby>確認<rt>かくにん</rt></ruby>させてください。」
   * (请让我确认一下分支策略。) —— 确认是 GitFlow 还是 GitHub Flow。
 * 「プルリクの<ruby>承認<rt>しょうにん</rt></ruby>をトリガーにデプロイされます。」
   * (以 PR 的通过为触发点进行部署。)

5. <ruby>重要<rt>じゅうよう</rt></ruby>！<ruby>証跡<rt>しょうせき</rt></ruby> (Evidence) - 证据管理
日本现场对“证据”的执着在发布环节达到顶峰：
 * <ruby>実行結果<rt>じっこうけっか</rt></ruby>ログ: Pipeline 的执行日志。
 * <ruby>正常終了<rt>せいじょうしゅうりょう</rt></ruby>のキャプチャ: * 部署成功的截图（通常要截到绿色的勾勾 ✅）。
 * <ruby>適用前後<rt>てきようぜんご</rt></ruby>の<ruby>比較<rt>ひかく</rt></ruby>: * 部署前后的对比（证明配置确实生效了）。

💡 给开发者的建议
 * 命名规范： 日本现场对资源的命名（Naming Convention）非常严格，写 Bicep/Terraform 前一定要先看文档里的命名规则。
 * README： Azure Repos 里的 README 尽量用日语写清楚「<ruby>環境構築手順<rt>かんきょうこうちくてじゅん</rt></ruby>」（环境搭建步骤），这样会显得你非常靠谱。
