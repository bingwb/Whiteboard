在日本的 IT 现场，Azure 的术语通常是 “英文原词（片假名化）+ 日语动词/名词” 的组合。日本开发者在交流时，非常喜欢将 Azure 的特定功能缩写或直接用片假名表达。
以下是为您整理的 Azure <ruby>関連<rt>かんれん</rt></ruby><ruby>用語集<rt>ようごしゅう</rt></ruby>。

☁️ Azure <ruby>関連<rt>かんれん</rt></ruby><ruby>常用<rt>じょうよう</rt></ruby><ruby>用語集<rt>ようごしゅう</rt></ruby>
1. <ruby>基本概念<rt>きほんがいねん</rt></ruby> - 基础概念

| 术语 | 片假名/日语 | 说明 |
|---|---|---|
| テナント | テナント | Tenant（租户），通常指整个组织实例。 |
| サブスクリプション | サブスク / サブスクリプション | Subscription（订阅），费用和资源的隔离单位。 |
| リソースグループ | リソースグループ (RG) | Resource Group，资源的逻辑容器。 |
| リージョン | リージョン | Region（地域），如「<ruby>東日本<rt>ひがしにほん</rt></ruby>リージョン」。 |
| 可用性セット | <ruby>可用性<rt>かようせい</rt></ruby>セット | Availability Set，可用性集。 |
| マネージドディスク | マネージドディスク | Managed Disk，托管磁盘。 |

2. <ruby>主要<rt>しゅよう</rt></ruby>サービス - 核心服务
这些词在讨论架构时每天都会用到：
 * Compute:
   * <ruby>仮想<rt>かそう</rt></ruby>マシン (VM): 虚拟机。日本现场极少说 Virtual Machine，全说「ブイエム」。
   * App Service: 常听到的术语是「アップサービス」。
   * Azure Functions: 常简称为「ファンクションズ」。常讨论「サーバーレス（无服务器）」。
   * AKS: Azure Kubernetes Service 的缩写。
 * Storage & DB:
   * ストレージアカウント: Storage Account。
   * BLOB: ブロブ。常说「BLOB（ブロブ）に<ruby>格納<rt>かくのう</rt></ruby>する」（存入 BLOB）。
   * Azure SQL Database: SQLデータベース。
   * Cosmos DB: コスモスディービー。
 * Networking:
   * VNet (Virtual Network): ブイネット。常讨论「VNet<ruby>統合<rt>とうごう</rt></ruby>（VNet 集成）」。
   * NSG (Network Security Group): エヌエスジー。讨论端口开放时必说。
   * <ruby>負荷分散装置<rt>ふかぶんさんそうち</rt></ruby>: Load Balancer。

3. <ruby>認証<rt>にんしょう</rt></ruby>・セキュリティ - 认证与安全
 * Microsoft Entra ID: (原 Azure AD)。现场老员工可能还会习惯叫「アジュールエーディー」。
 * <ruby>条件付<rt>じょうけんつ</rt></ruby>きアクセス: Conditional Access（条件访问）。
 * サービスプリンシパル: Service Principal（服务主体）。在做自动化部署时常用。
 * マネージドID: Managed Identity。
 * キーコンテナー: Key Vault。用于存储密码和证书。

4. <ruby>構築<rt>こうちく</rt></ruby>・<ruby>運用<rt>うんよう</rt></ruby> - 搭建与运维
 * ポータル: Azure Portal 界面。
 * ARMテンプレート: Azure Resource Manager 模板。
 * Bicep: バイセップ。现在主流的 IaC 语言。
 * コマンドライン (CLI): アジュールシーエルアイ。
 * <ruby>監視<rt>かんし</rt></ruby> (Monitor): Azure Monitor。常讨论「メトリック（指标）」和「ログ（日志）」。
 * アラート<ruby>設定<rt>せってい</rt></ruby>: Alert 设置。

5. <ruby>現場<rt>げんば</rt></ruby>でよく<ruby>使<rt>つか</rt></ruby>われるフレーズ - Azure 相关常用语
 * 「<ruby>権限<rt>けんげん</rt></ruby>を<ruby>付与<rt>ふよ</rt></ruby>してください。」
   * (请授予我权限。) —— 比如需要某个 RG 的 Contributor 权限时。
 * 「NSGの<ruby>設定<rt>せってい</rt></ruby>を<ruby>確認<rt>かくにん</rt></ruby>してください。」
   * (请确认 NSG 的设置。) —— 网络不通时第一句必说的话。
 * 「リソースをデプロイしました。」
   * (资源已经部署好了。)
 * 「コストが<ruby>高<rt>たか</rt></ruby>いので、サイズを<ruby>縮小<rt>しゅくしょう</rt></ruby>します。」
   * (因为成本太高，我们要缩小实例规格。) —— 这里的“规格”常说成「インスタンスサイズ」。
6. <ruby>注意<rt>ちゅうい</rt></ruby>すべき<ruby>和製英語<rt>わせいえいご</rt></ruby> / <ruby>略語<rt>りゃくご</rt></ruby>
 * デプロイ: Deploy（部署）。
 * サイジング: Sizing。决定 VM 或 DB 的规格。
 * スナップショット: Snapshot（快照）。
 * 立ち上げる（たちあげる）: 启动（服务/实例）。

💡 总结建议
在 Azure 现场，建议您随身准备好 Azure Portal 的日语/英语对照。因为很多日本现场的服务器 OS 是日语版的，Portal 也是日语版的，熟悉「<ruby>作成<rt>さくせい</rt></ruby> (Create)」、「<ruby>編集<rt>へんしゅう</rt></ruby> (Edit)」、「<ruby>削除<rt>さくじょ</rt></ruby> (Delete)」等基础词汇非常重要。
