作为在日本工作的开发者，掌握这些术语不仅能帮你听懂同事在说什么，还能让你在写文档时显得非常专业。
我为你整理了一份**《日本 IT 现场高频术语集》**，涵盖了工程、技术、沟通以及职场俚语，并使用了 <ruby> 标签和 Markdown 表格。

1. <ruby>開発工程<rt>かいはつこうてい</rt></ruby> - 开发阶段
日本现场通常遵循 V 字模型，每个阶段都有固定缩写。

| 术语 | 读音 | 含义 | 备注 |
|---|---|---|---|
| <ruby>要件定義<rt>ようけんていぎ</rt></ruby> | RD | 需求定义 | Requirement Definition |
| <ruby>基本設計<rt>きほんせっけい</rt></ruby> | BD / FD | 基本设计 | 功能设计 (Functional Design) |
| <ruby>詳細設計<rt>しょうさいせっけい</rt></ruby> | DD | 详细设计 | Detail Design |
| <ruby>製造<rt>せいぞう</rt></ruby> / <ruby>実装<rt>じっそう</rt></ruby> | Coding | 编码/实装 | 日本现场常把 Coding 叫“制造” |
| <ruby>単体<rt>たんたい</rt></ruby>テスト | UT | 单体测试 | Unit Test |
| <ruby>結合<rt>けつごう</rt></ruby>テスト | IT | 结合测试 | Integration Test (分为 ITa/ITb) |
| <ruby>総合<rt>そうごう</rt></ruby>テスト | ST | 综合测试 | System Test |
| <ruby>受入<rt>うけいれ</rt></ruby>テスト | UAT | 验收测试 | User Acceptance Test |
| <ruby>保守運用<rt>ほしゅうんよう</rt></ruby> | - | 运维保养 | Maintenance & Operations |

2. <ruby>技術・実装関連<rt>ぎじゅつ・じっそうかんれん</rt></ruby> - 技术与实装
这些词在 Code Review 或讨论逻辑时频繁出现。
 * <ruby>仕様<rt>しよう</rt></ruby> (Specification): 规格/需求。最常说：“那是<ruby>仕様<rt>しよう</rt></ruby>（需求如此，不是Bug）”。
 * <ruby>考慮漏<rt>こうりょも</rt></ruby>れ (Missing Consideration): 考虑不周/逻辑漏洞。
 * <ruby>先祖返<rt>せんぞがえ</rt></ruby>り (Regression): 代码回滚/旧Bug复现。
 * <ruby>不具合<rt>ふぐあい</rt></ruby> / バグ: Bug/故障。日本现场正式文档多用「不具合」。
 * デグレ (Degradation): 性能下降或由于改动导致原有功能失效。
 * <ruby>排他制御<rt>はいたせいぎょ</rt></ruby> (Locking/Concurrency Control): 互斥控制/锁。
 * <ruby>共通化<rt>きょうつうか</rt></ruby> (Refactoring to common): 提取公用组件/代码复用。
 * <ruby>直書<rt>じかが</rt></ruby>き / ハードコーディング: 硬编码。

3. インフラ・<ruby>環境<rt>かんきょう</rt></ruby> - 基础设施与环境
 * <ruby>本番環境<rt>ほんばんかんきょう</rt></ruby>: 生产环境 / PROD。
 * <ruby>検証環境<rt>けんしょうかんきょう</rt></ruby>: 验证环境 / STG。
 * <ruby>開発環境<rt>かいはつかんきょう</rt></ruby>: 开发环境 / DEV。
 * ローカル<ruby>環境<rt>かんきょう</rt></ruby>: 本地环境。
 * <ruby>踏<rt>ふ</rt></ruby>み<ruby>台<rt>だい</rt></ruby>サーバ: 跳板机 / Bastion Server。
 * <ruby>疎通確認<rt>そつうかくにん</rt></ruby>: 网络打通确认（Ping/Telnet测试等）。

4. <ruby>書類・納品物<rt>しょるい・のうひんぶつ</rt></ruby> - 文档与交付物
日本 IT 现场是文档驱动的，这些词必须知道：
 * エビデンス (Evidence): 证据。通常指测试截图。
 * <ruby>設計書<rt>せっけいしょ</rt></ruby>: 设计书。
 * <ruby>手順書<rt>てじゅんしょ</rt></ruby>: 操作手册/步骤说明（如发布步骤）。
 * <ruby>成果物<rt>せいかぶつ</rt></ruby>: 交付成果。
 * <ruby>差分<rt>さぶん</rt></ruby>: Diff / 差异。
 * <ruby>雛形<rt>ひながた</rt></ruby>: 模板。

5. <ruby>現場<rt>げんば</rt></ruby>のスラング・<ruby>和製英語<rt>わせいえいご</rt></ruby> - 现场俚语
这些词具有浓厚的日本 IT 职场特色：
 * リスケ (Reschedule): 重新安排计划/延期。
 * アサイン (Assign): 指派任务。“我被<ruby>アサイン<rt>あさいん</rt></ruby>到了这个项目”。
 * バッファ (Buffer): 缓冲时间。“给这个任务留 2 天的<ruby>バッファ<rt>ばっふぁ</rt></ruby>”。
 * ボールがある: 球在谁手里（谁负责下一步）。“现在<ruby>ボール<rt>ぼーる</rt></ruby>在客户那边”。
 * たたき<ruby>台<rt>だい</rt></ruby>: 草案/初稿。
 * <ruby>握<rt>にぎ</rt></ruby>る: 达成一致/确认。“跟客户<ruby>握<rt>にぎ</rt></ruby>っておきました”。
 * フィックス (Fix): 定稿/确定。“逻辑已经<ruby>フィックス<rt>ふぃっくす</rt></ruby>了”。
 * <ruby>有識者<rt>ゆうしきしゃ</rt></ruby>: 专家/懂行的人。遇到不懂的逻辑要问“现场的<ruby>有識者<rt>ゆうしきしゃ</rt></ruby>”。
 * ポシャる: 项目流产/计划告吹。

6. <ruby>会議・報告<rt>かいぎ・ほうこく</rt></ruby> - 会议与汇报
 * <ruby>朝会<rt>ちょうかい</rt></ruby>: 早会（Daily Standup）。
 * <ruby>定例会<rt>ていれいかい</rt></ruby>: 定期周会/例会。
 * キックオフ: 项目启动会 (Kick-off)。
 * <ruby>周知<rt>しゅうち</rt></ruby>: 通知/周知。“全员に<ruby>周知<rt>しゅうち</rt></ruby>してください”。
 * <ruby>展開<rt>てんかい</rt></ruby>: 转发/散播。“这个邮件请向团队内<ruby>展開<rt>てんかい</rt></ruby>”。

7. <ruby>勤怠・一般<rt>きんたい・いっぱん</rt></ruby> - 考勤与一般
 * <ruby>稼働<rt>かどう</rt></ruby>: 工时/劳动量。“最近<ruby>稼働<rt>かどう</rt></ruby>が高いですね（最近加班挺多）”。
 * <ruby>定時<rt>ていじ</rt></ruby>: 准时下班。
 * <ruby>全休<rt>ぜんきゅう</rt></ruby> / <ruby>半休<rt>はんきゅう</rt></ruby>: 全天请假 / 半天假。
 * <ruby>中抜<rt>なかぬ</rt></ruby>け: 工作时间临时外出（如看病、接孩子）。

💡 学习建议
这些词不需要死记硬背。你可以：
 * 看 Wiki/Slack: 看看同事在讨论问题时喜欢用哪个词。
 * 写文档: 尽量模仿 Leader 使用的术语。
 * Excel 搜索: 在日本现场，很多逻辑都埋在 Excel 里，搜索这些关键词（如「考慮漏れ」）能帮你快速找到历史记录。
